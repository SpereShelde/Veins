Detailed Description

The basic Decider class.

The Deciders tasks are:

  1. decide which packets should be handed up to the MAC Layer (primary task)
  2. decide whether the channel is busy/idle at a time point or during a time interval (channel sensing)

BasePhyLayer hands every receiving AirFrame several times to the "processSignal()"-function and is returned a time point when to do so again.

◆ switchToTx()

virtual void veins::Decider::switchToTx ( )
inlinevirtual

Notifies the decider that phy layer is starting a transmission.

This helps the decider interrupting a current reception. In a standard 802.11 MAC, this should never happen, but in other MAC layers you might decide to interrupt an ongoing reception and start transmitting. Thank to this method, the decider can flag the ongoing frame as non received because of the transmission.
