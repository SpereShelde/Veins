74  /** @brief Power level threshold used to declare channel busy if
75  * preamble portion is missed (802.11-2012 18.3.10.6
76  * CCA requirements). Notice that in 18.3.10.6, the mandatory CCA threshold
77  * for a 10 MHz channel is -65 dBm. However, there is another threshold
78  * called CCA-ED which requirements are defined in D.2.5. For a 10 MHz
79  * channel, CCA-ED threshold shall be -75 dBm. CCA-ED is required for
80  * certain operating classes that shall implement CCA-ED behavior.
81  * According to Table E-4 however, 802.11p channels should not implement
82  * it, so the correct threshold is -65 dBm.
83  * When considering ETSI ITS G5 (ETSI TS 102 687) things change again.
84  * Indeed, the DCC Sensitivity Control (DSC) part of the DCC algorithm
85  * changes the CCA threshold depending on the state. Values are listed
86  * in Table A.3: minimum value is -95 dBm, maximum value is -65 dBm,
87  * and default value is -85 dBm.
88  */
89  double ccaThreshold;
90 
91  /** @brief allows/disallows interruption of current reception for txing
92  *
93  * For a standard 802.11 MAC, starting a transmission while currently
94  * receiving a frame is forbidden, as the channel is in a BUSY state.
95  * For research purposes, however, one might use a custom MAC layer on
96  * top of an 802.11p OFDM PHY, and decide to interrupt an ongoing
97  * reception to start a transmission, whatever the reason. If the
98  * following variable is set to false, simulation will be terminated
99  * if this occurs. If not, simulation will continue, aborting current
100  * reception.
101  */
102  bool allowTxDuringRx;
