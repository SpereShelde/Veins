Compare two floats for approximate equality.

Floating point numbers are prone to inequalities. Expressions that logically result in the same values therefore often are not exactly equal, because two representations of very similar values are computed. This function allows comparisons for such cases by allowing for a small error (epsilon), without considering numbers to be unequal.

Parameters
xFirst comparand
ySecond comparand
ulpHow many ULP's (Units in the Last Place) we want to tolerate when comparing two numbers. The larger the value, the more error we allow. The maximum floating point error on most platforms is below 0.5 ULP. A 0 value means that two numbers must be exactly the same to be considered equal.
Returns
Indicator whether the lhs and rhs are virtually equal
